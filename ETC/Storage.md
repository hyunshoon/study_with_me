## 스토리지 정리

블록 스토리지는 작은 단위의 데이터를 빠르게 처리하며, 파일 스토리지는 파일 단위의 데이터를 처리하고 공유하는 데 효과적입니다. 객체 스토리지는 대용량 파일을 효율적으로 처리하고, 복제 및 백업을 위한 데이터 보존에 적합

### 블록 스토리지

```
블록 단위로 데이터를 저장하는 스토리지 방식. 각 블록은 고유한 주소를 가지며, 이를 통해 데이터를 읽거나 쓸 수 있다.
주로 서버 가상화, 디스크 저장소, DB 등의 용도로 사용.

스토리지 영역과 서버 간에 네트워크 연결을 통해 접근 가능.
블록 스토리지의 장점으로는 빠른 응답 시간과 높은 성능, 안정성, 가용성이 있다.

블록 단위로 데이터를 관리하기 때문에 일부만 변경되더라도 해당 블록만 변경하면 되기 때문에 데이터 전체를 다시 저장할 필요가 없다.
또한, 블록 단위의 I/O 작업만 수행하기 때문에 데이터 처리 속도가 빠르다.

```

### 파일 스토리지
```
파일 단위로 데이터를 저장. 파일 시스템을 기반으로 동작하며, 디렉토리 구조와 파일이름으로 데이터를 관리.
대용량 파일의 처리와 공유 파일 시스템 등의 용도로 주로 사용
```

### 객체 스토리지
```
객체 단위로 데이터를 저장. 각 객체는 고유한 ID를 가지며, 이를 통해 데이터를 관리한다. 대용량 파일의 처리와 복제, 백업 등에 적합.
대규모 데이터 처리를 지원하며, 저장된 데이터의 메타데이터를 활용하여 데이터 검색과 관리에 용이.
객체 스토리지는 주로 대용량 파일 스토리지, 웹 콘텐츠 스토리지 등의 용도로 사용된다.
```
